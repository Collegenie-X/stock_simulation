## 전체 유저 플로우

```mermaid
flowchart TD
    A[게임 시작] --> B[튜토리얼]
    B --> C{튜토리얼<br/>건너뛰기?}
    C -->|예| D[게임 설정]
    C -->|아니오| E[튜토리얼 진행]
    E --> D
    
    D --> F[초기 자본 선택]
    F --> G[기간 선택]
    G --> H[게임 메인 화면]
    
    H --> I{액션 선택}
    I -->|일시정지| J[10초 타이머 발동]
    I -->|관망| K[시간 진행]
    I -->|빨리감기| L[2배속/5배속]
    
    J --> M[매수/매도 결정]
    M --> N{시간 내<br/>결정?}
    N -->|예| O[거래 체결]
    N -->|아니오| P[자동 재개]
    
    O --> H
    P --> H
    K --> H
    L --> H
    
    H --> Q{기간<br/>종료?}
    Q -->|아니오| I
    Q -->|예| R[결과 리포트]
    
    R --> S[투자 성향 분석]
    S --> T[랭킹 확인]
    T --> U{다시<br/>시작?}
    U -->|예| D
    U -->|아니오| V[게임 종료]
```

## 시나리오 1: 첫 게임 시작

### 1.1 시작 화면

| 단계 | 사용자 행동 | 시스템 반응 | 화면 요소 |
|------|------------|------------|----------|
| 1 | 앱 실행 | 메인 화면 표시 | 로고, [시작하기] 버튼 |
| 2 | [시작하기] 클릭 | 튜토리얼 팝업 표시 | "처음이신가요?" 메시지 |
| 3 | [예] 또는 [건너뛰기] 선택 | 선택에 따라 분기 | 튜토리얼 or 설정 화면 |

```mermaid
sequenceDiagram
    participant U as 사용자
    participant S as 시스템
    participant UI as 화면
    
    U->>S: 앱 실행
    S->>UI: 메인 화면 렌더링
    UI->>U: 시작 버튼 표시
    U->>S: [시작하기] 클릭
    S->>UI: 튜토리얼 팝업
    U->>S: [예] 선택
    S->>UI: 튜토리얼 1단계 표시
```

### 1.2 튜토리얼 진행

| 단계 | 내용 | 소요 시간 | 학습 목표 |
|------|------|----------|----------|
| 1 | 게임 목표 설명 | 10초 | 시뮬레이션 목적 이해 |
| 2 | 화면 구성 소개 | 15초 | UI 요소 파악 |
| 3 | 시간 시스템 설명 | 20초 | 1일=1분, 4단계 구조 이해 |
| 4 | 일시정지 & 10초 타이머 | 30초 | 핵심 메커니즘 체험 |
| 5 | 매수/매도 실습 | 40초 | 실제 거래 연습 |
| 6 | 종목 카테고리 안내 | 15초 | 안정/변동/고변동 이해 |

```mermaid
stateDiagram-v2
    [*] --> 목표설명
    목표설명 --> 화면구성
    화면구성 --> 시간시스템
    시간시스템 --> 일시정지체험
    일시정지체험 --> 매수실습
    매수실습 --> 매도실습
    매도실습 --> 종목소개
    종목소개 --> [*]
    
    note right of 일시정지체험
        10초 타이머 발동
        실제 체험
    end note
```

### 1.3 게임 설정

| 설정 항목 | 옵션 | 설명 |
|----------|------|------|
| **초기 자본** | 500만원 | 초보자 추천 ⭐ |
| | 1,000만원 | 일반적인 시작 |
| | 5,000만원 | 여유로운 플레이 |
| | 1억원 | 다양한 분산 투자 |
| | 5억원 | 대규모 전략 |
| | 10억원 | 고급 사용자용 |
| **기간** | 3개월 (90일) | 빠른 체험 (1.5시간) ⭐ |
| | 6개월 (180일) | 균형잡힌 체험 (3시간) |
| | 12개월 (365일) | 장기 전략 (6시간) |

### 2.2 시간 진행 시나리오

| 시간대 | 실제 경과 시간 | 게임 내 시간 | 발생 이벤트 | 사용자 가능 행동 |
|--------|---------------|------------|-----------|----------------|
| 09:00 | 0초 | 시초가 | 시가 결정, 뉴스 표시 | 종목 확인, 일시정지 |

## 시나리오 3: 매매 의사결정

### 3.1 일시정지 및 10초 타이머

```mermaid
sequenceDiagram
    participant U as 사용자
    participant G as 게임
    participant T as 타이머
    participant M as 시장
    
    U->>G: [일시정지] 클릭
    G->>M: 시간 정지
    G->>T: 10초 타이머 시작
    T->>U: 카운트다운 표시 (10...9...8...)
    
    alt 10초 이내 결정
        U->>G: [매수] 클릭
        G->>M: 현재 가격으로 체결
        M->>U: 체결 완료 알림
        G->>T: 타이머 중지
        G->>M: 시간 재개
    else 10초 초과
        T->>G: 타이머 종료
        G->>M: 자동 시간 재개
        M->>U: "시간 초과, 가격 변동됨" 알림
    end
```

### 3.2 매수 프로세스

| 단계 | 사용자 행동 | 시스템 체크 | 결과 |
|------|------------|------------|------|
| 1 | 종목 선택 | - | 차트 표시 |
| 2 | [매수] 버튼 클릭 | 일시정지 여부 확인 | 10초 타이머 발동 |
| 3 | 수량 입력 | 현금 잔액 확인 | 가능 수량 표시 |
| 4 | [확정] 클릭 | 수수료 계산 | 체결 또는 오류 |
| 5 | - | 포트폴리오 업데이트 | 보유 종목 추가 |

```mermaid
flowchart TD
    A[종목 선택] --> B{일시정지<br/>상태?}
    B -->|아니오| C[일시정지 자동 실행]
    B -->|예| D[매수 팝업]
    C --> D
    
    D --> E[10초 타이머 시작]
    E --> F[수량 입력]
    F --> G{현금<br/>충분?}
    G -->|아니오| H[오류 메시지:<br/>현금 부족]
    G -->|예| I[예상 금액 표시]
    
    I --> J{10초<br/>남음?}
    J -->|아니오| K[자동 취소]
    J -->|예| L[확정 버튼 클릭]
    
    L --> M[체결 처리]
    M --> N[효과음: 매수 완료]
    N --> O[포트폴리오 업데이트]
    O --> P[시간 재개]
    
    H --> Q[다시 입력]
    Q --> F
    K --> R[알림: 시간 초과]
    R --> P
```

### 3.3 매도 프로세스

```mermaid
flowchart TD
    A[보유 종목 선택] --> B[현재 수익률 표시]
    B --> C[매도 버튼 클릭]
    C --> D[일시정지 & 10초 타이머]
    D --> E[매도 수량 선택]
    E --> F{전량<br/>매도?}
    F -->|예| G[전체 선택]
    F -->|아니오| H[일부 수량 입력]
    
    G --> I[예상 수령액 표시]
    H --> I
    I --> J[수수료 + 거래세 표시]
    J --> K[순수익 계산 표시]
    K --> L{10초<br/>내 확정?}
    L -->|예| M[체결]
    L -->|아니오| N[취소]
    
    M --> O[현금 증가]
    O --> P[보유 종목 감소/제거]
    P --> Q{수익?}
    Q -->|예| R[효과음: 수익 축하]
    Q -->|아니오| S[효과음: 손실]
    R --> T[거래 기록 저장]
    S --> T
    T --> U[시간 재개]
    
    N --> U
```

### 3.4 의사결정 시간별 패턴

| 결정 시간 | 비율 | 투자 성향 | 결과 경향 |
|----------|------|---------|----------|
| 0-3초 (즉시) | 38% | 충동적, 직관형 | 승률 낮음 (45%) |
| 4-6초 (빠름) | 35% | 적극적, 경험형 | 승률 중간 (62%) |
| 7-9초 (신중) | 22% | 분석적, 신중형 | 승률 높음 (74%) |
| 10초 (시간초과) | 5% | 우유부단 | 기회 손실 |

## 시나리오 4: 게임 종료

## 페이지별 상세 시나리오

### Page 1: 메인 화면

| 요소 | 기능 | 사용자 액션 |
|------|------|-----------|
| 로고 | 브랜딩 | - |
| [시작하기] | 게임 시작 | 클릭 → 튜토리얼/설정 |
| [이어하기] | 저장된 게임 불러오기 | 클릭 → 게임 로드 |
| [랭킹] | 전체 랭킹 보기 | 클릭 → 랭킹 화면 |
| [설정] | 소리, 효과 설정 | 클릭 → 설정 화면 |

```mermaid
flowchart LR
    A[설정 화면] --> B[초기 자본 선택]

    B2 --> D
    B3 --> D
    B4 --> D
    B5 --> D
    B6 --> D
```

### Page 4: 매매 팝업

| 요소 | 정보 | 입력/출력 |
|------|------|----------|
| 타이머 | 10초 카운트다운 | 출력 (시각적) |
