# Part 1: 타이밍 계산 기초

## 1-1. 결정 1회의 시간 해부

모든 시간 계산의 기본 단위: **"결정 1회 (1 Decision Cycle)"**

```
결정 1회 = 카드 등장 + 읽기/판단 + 실행 + 피드백 + 전환

이 사이클이 모든 계산의 기초입니다.
```

### 모드별 결정 1회 소요 시간

| 요소 | ⚡ 스프린트 | 🏃 스탠다드 | 🏔️ 마라톤 |
|------|-----------|-----------|----------|
| (a) 카드 등장 애니메이션 | 0.5초 | 0.8초 | 1.0초 |
| (b) 타이머 최대값 | **10초** | **15초** | **20초** |
| (c) 평균 사용 시간 (타이머의 ~70%) | **7초** | **10초** | **14초** |
| (d) 실행 애니메이션 | 0.3초 | 0.5초 | 0.5초 |
| (e) 피드백 표시 | 1.5초 | 2.5초 | 4.0초 |
| (f) 다음 전환 | 0.3초 | 0.3초 | 0.5초 |
| **합계 (평균)** | **9.6초 ≈ 10초** | **14.1초 ≈ 14초** | **20.0초** |
| **합계 (최대)** | **12.6초 ≈ 13초** | **19.1초 ≈ 19초** | **26.0초** |

> **핵심**: 평균 플레이어 기준으로 계산. 느린 플레이어는 시간이 더 걸림.

## 1-2. 결정 외 시간 요소

| 요소 | ⚡ 스프린트 | 🏃 스탠다드 | 🏔️ 마라톤 |
|------|-----------|-----------|----------|
| **시작 화면 (모드 선택, 설정)** | 5초 | 8초 | 12초 |
| **시간대 전환 (아침→저녁 등)** | 0.3초 | 0.5초 | 0.5초 |
| **자동 경과 (이벤트 없는 날)** | 0.8초/일 | 0.8초/일 | 0.8초/일 |
| **자동 경과 (묶음 3일+)** | 1.5초 | 1.5초 | 1.5초 |
| **주간 미니 리포트** | 3초 | 5초 | 8초 |
| **월간 리포트** | - | 8초 | 12초 |
| **최종 리포트** | 10초 | 15초 | 20초 |
| **위기 이벤트 (특수)** | 12초 | 18초 | 25초 |

## 1-3. "이벤트 날" vs "평범한 날"

```
핵심 설계: 모든 날을 플레이하지 않는다!

20 거래일 (1개월) 중:
- 이벤트 날: 실적 발표, 뉴스, 급등락, 기회 → 결정 필요
- 평범한 날: 시장 안정, 큰 변동 없음 → 자동 경과 (0.8~1.5초)

이벤트 발생 확률: 약 50~60% (모드별 조정)
- 스프린트: 55% (20일 중 11일 이벤트)
- 스탠다드: 50% (60일 중 30일 이벤트)
- 마라톤: 45% (240일 중 108일 이벤트)
```

| ⚡ 스프린트 | 22회 | 5분 | **4.4회/분** | 듀오링고 수준 |
| 🏃 스탠다드 | 33회 | 10분 | **3.3회/분** | 적절한 밀도 |
| 🏔️ 마라톤 | 66회 | 30분 | **2.2회/분** | 전략 게임 수준 |

```
핵심: 분당 2~5회 결정 = "빠르다"는 느낌

+ 결정 사이 자동 경과 (1~3초)로 "시간이 흐르는" 느낌
+ 위기 이벤트로 "긴장감 급상승" 순간
+ 콤보/HP로 "한 순간도 방심할 수 없는" 느낌

= 적은 결정이지만 체감 밀도는 높음!
```

## 8-3. 자동 경과 = "숨 쉬는 시간"

```
결정만 연속하면 피로합니다.
자동 경과는 "숨 쉬는 시간" 역할:

[결정] → [피드백] → [자동경과 1초] → [결정] → ...

이 리듬이 핵심!

좋은 리듬: 🎵
  긴장(결정) → 이완(피드백) → 휴식(경과) → 긴장(결정)

나쁜 리듬: 😵
  결정 → 결정 → 결정 → 결정 (지침)
  
  또는
  
  경과 → 경과 → 경과 → 결정 (지루함)
```
